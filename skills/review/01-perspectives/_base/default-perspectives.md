# Review 視角定義

本 Skill 的視角定義集中管理於 `shared/perspectives/catalog.yaml`。

## 引用方式

```yaml
# 從 catalog.yaml 載入本 skill 適用的視角
perspectives:
  source: shared/perspectives/catalog.yaml
  filter:
    category: review
  preset: standard  # 預設使用 4 視角
```

## 視角列表

### 標準模式 (4 視角)

| ID | 名稱 | 關注重點 | 模型層級 | 方法 |
|----|------|----------|----------|------|
| `code-quality` | 程式碼品質審查員 | 風格一致性和設計模式 | haiku | explore |
| `test-coverage` | 測試覆蓋審查員 | 測試品質和邊界案例 | haiku | explore |
| `documentation` | 文檔審查員 | API 文檔和註解品質 | haiku | explore |
| `integration` | 整合審查員 | 向後相容和 API 契約 | sonnet | explore |

### 快速模式 (2 視角)

| ID | 名稱 | 關注重點 |
|----|------|----------|
| `code-quality` | 程式碼品質審查員 | 風格一致性和設計模式 |
| `integration` | 整合審查員 | 向後相容和 API 契約 |

### 深度模式 (6 視角)

標準 4 視角 + 跨領域視角：

| ID | 名稱 | 關注重點 |
|----|------|----------|
| `security-auditor` | 安全審計員 | OWASP 和安全最佳實踐 |
| `compliance-checker` | 合規檢查員 | 法規遵循和隱私保護 |

## 視角協作模式

```
┌─────────────────────────────────────────────────────────┐
│                    程式碼變更                            │
├─────────────────────────────────────────────────────────┤
│  品質層                          整合層                  │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐│
│  │ 程式碼   │  │ 測試     │  │ 文檔     │  │ 整合     ││
│  │ 品質     │  │ 覆蓋     │  │ 審查     │  │ 審查     ││
│  │          │  │          │  │          │  │          ││
│  │ 風格一致 │  │ 測試品質 │  │ API 文檔 │  │ 向後相容 ││
│  │ 設計模式 │  │ 邊界案例 │  │ 註解完整 │  │ 依賴影響 ││
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘│
└─────────────────────────────────────────────────────────┘
```

## 視角組合策略

| 場景 | 推薦組合 | 說明 |
|------|----------|------|
| 快速審查 | code-quality + test-coverage | 核心品質 + 測試保證 |
| 安全敏感 | code-quality + integration | 品質保證 + 整合安全 |
| 文檔審查 | documentation + integration | 文檔完整 + API 相容 |
| 全面審查 | 全部 4 視角 | 預設模式 |

## 審查輸出格式

各視角產出包含：
- 問題列表（含行號、嚴重度、建議）
- 優點列表
- 改進建議

完整定義請參考 [catalog.yaml](../../../../shared/perspectives/catalog.yaml)。
