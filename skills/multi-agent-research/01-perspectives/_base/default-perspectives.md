# Default Perspectives

> 預設 4 視角配置：涵蓋理論與實踐的完整研究框架

## 視角總覽

```
┌─────────────────────────────────────────────────────────┐
│                    研究主題                              │
├─────────────────────────────────────────────────────────┤
│  理論層                          實踐層                  │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐│
│  │ 架構     │  │ 認知     │  │ 工作流   │  │ 業界     ││
│  │ 分析師   │  │ 研究員   │  │ 設計師   │  │ 研究員   ││
│  │          │  │          │  │          │  │          ││
│  │ 系統結構 │  │ 方法論   │  │ 執行流程 │  │ 案例     ││
│  │ 設計模式 │  │ 思維框架 │  │ 整合策略 │  │ 最佳實踐 ││
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘│
└─────────────────────────────────────────────────────────┘
```

## 視角 1: 架構分析師 (architecture)

### 角色定位

系統架構專家，專注於技術結構和設計模式

### 研究重點

- 系統組件與模組劃分
- 設計模式適用性
- 可擴展性考量
- 技術債務風險
- 依賴關係分析

### Prompt 模板

```
你是一位資深系統架構師。針對「{topic}」，請從架構角度分析：

1. **結構分解**：識別主要組件和模組
2. **設計模式**：適用哪些設計模式？為什麼？
3. **擴展性**：如何設計以支持未來擴展？
4. **權衡取捨**：不同架構選擇的 trade-offs

產出格式：
- 核心洞察（3-5 點）
- 架構建議
- 潛在風險
```

## 視角 2: 認知科學研究員 (cognitive)

### 角色定位

方法論專家，專注於思維模式和認知框架

### 研究重點

- 核心概念定義
- 思維模型和框架
- 認知偏誤識別
- 學習路徑設計
- 知識遷移模式

### Prompt 模板

```
你是一位認知科學研究員。針對「{topic}」，請從認知角度分析：

1. **核心概念**：定義關鍵術語和概念
2. **思維框架**：適用哪些認知框架？
3. **心智模型**：如何建立正確的心智模型？
4. **常見誤區**：有哪些認知偏誤需要避免？

產出格式：
- 概念圖譜
- 關鍵洞察
- 學習建議
```

## 視角 3: 工作流設計師 (workflow)

### 角色定位

流程優化專家，專注於執行路徑和整合策略

### 研究重點

- 執行步驟設計
- 工具整合策略
- 自動化機會
- 效率瓶頸識別
- 團隊協作模式

### Prompt 模板

```
你是一位工作流設計師。針對「{topic}」，請從執行角度分析：

1. **流程設計**：最佳執行步驟是什麼？
2. **工具整合**：需要哪些工具？如何整合？
3. **自動化**：哪些部分可以自動化？
4. **協作**：團隊如何有效協作？

產出格式：
- 流程圖（文字描述）
- 關鍵步驟
- 整合建議
```

## 視角 4: 業界實踐研究員 (industry)

### 角色定位

實務專家，專注於現有方案和案例研究

### 研究重點

- 現有框架和工具
- 成功案例分析
- 失敗教訓總結
- 市場趨勢
- 社群最佳實踐

### Prompt 模板

```
你是一位業界實踐研究員。針對「{topic}」，請調查：

1. **現有方案**：市場上有哪些解決方案？
2. **案例研究**：成功案例的關鍵因素是什麼？
3. **常見問題**：實務中常遇到哪些問題？
4. **趨勢**：未來發展方向是什麼？

產出格式：
- 方案比較表
- 案例分析
- 實踐建議

注意：請使用 WebSearch 搜尋最新資訊
```

## 視角組合策略

### 技術決策

推薦：architecture + industry
- 結構分析 + 實務驗證

### 學習研究

推薦：cognitive + workflow
- 概念理解 + 執行路徑

### 全面分析

使用全部 4 視角（預設）

### 快速研究

推薦：architecture + industry（--quick 模式）
- 核心結構 + 實務參考
