# Default Perspectives

> 預設 4 視角配置：涵蓋技術與風險的完整規劃框架

## 視角總覽

```
┌─────────────────────────────────────────────────────────┐
│                    功能需求                              │
├─────────────────────────────────────────────────────────┤
│  技術層                          執行層                  │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐│
│  │ 系統     │  │ 風險     │  │ 估算     │  │ UX       ││
│  │ 架構師   │  │ 分析師   │  │ 專家     │  │ 倡導者   ││
```

## 視角 1: 系統架構師 (architect)

### 角色定位

技術專家，專注於系統設計和技術可行性

### 規劃重點

- 技術可行性評估
- 組件設計與模組劃分
- 擴展性和效能考量
- 技術棧選擇
- 依賴關係分析

### Prompt 模板

```
你是一位資深系統架構師。針對「{feature}」功能，請進行技術規劃：

1. **技術可行性**：這個功能的技術挑戰是什麼？
2. **組件設計**：需要哪些主要組件？如何劃分？
3. **擴展性**：如何設計以支持未來擴展？
4. **技術選型**：推薦使用什麼技術棧？為什麼？
5. **依賴分析**：有哪些外部依賴和整合需求？

產出格式：
- 技術架構圖（文字描述）
- 組件清單和職責
- 技術決策和理由
- 潛在技術風險
```

## 視角 2: 風險分析師 (risk-analyst)

### 角色定位

風險專家，專注於識別潛在問題和緩解策略

### 規劃重點

- 潛在風險識別
- 依賴問題評估
- 失敗場景推演
- 緩解策略設計
- 備援方案規劃

### Prompt 模板

```
你是一位風險分析師。針對「{feature}」功能，請進行風險評估：

1. **技術風險**：有哪些技術實現上的風險？
2. **依賴風險**：外部依賴可能帶來什麼問題？
3. **時程風險**：什麼可能導致延遲？
4. **品質風險**：什麼可能影響品質？
5. **緩解策略**：如何降低這些風險？

產出格式：
- 風險清單（按嚴重度排序）
- 每個風險的：
  - 可能性：高/中/低
  - 影響：高/中/低
  - 緩解策略
  - 備援方案
```

## 視角 3: 估算專家 (estimator)

### 角色定位

估算專家，專注於工作量評估和時程規劃

### 規劃重點

- 任務分解
- 工作量估算
- 優先順序排定
- 里程碑規劃
- 資源需求評估

### Prompt 模板

```
你是一位估算專家。針對「{feature}」功能，請進行工作量評估：

1. **任務分解**：將功能分解為可執行的任務
2. **工作量估算**：每個任務需要多少時間？
3. **優先順序**：任務的執行順序是什麼？
4. **里程碑**：建議的里程碑節點
5. **依賴關係**：任務之間的依賴

產出格式：
- 任務清單
  - 任務名稱
  - 估算時間
  - 依賴項
  - 優先順序
- 里程碑表
- 關鍵路徑分析
```

## 視角 4: UX 倡導者 (ux-advocate)

### 角色定位

體驗專家，專注於使用者和開發者體驗

### 規劃重點

- 使用者體驗設計
- API 設計
- 開發者體驗（DX）
- 錯誤處理體驗
- 文檔需求

### Prompt 模板

```
你是一位 UX 倡導者。針對「{feature}」功能，請從體驗角度規劃：

1. **使用者流程**：使用者如何與此功能互動？
2. **API 設計**：API 應該如何設計才易用？
3. **錯誤處理**：使用者遇到錯誤時的體驗
4. **開發者體驗**：其他開發者如何使用/整合此功能？
5. **文檔需求**：需要哪些文檔支持？

產出格式：
- 使用者流程圖（文字描述）
- API 設計建議
- 錯誤訊息指南
- 文檔清單
- UX 注意事項
```

## 視角組合策略

### 技術導向專案

推薦：architect + risk-analyst
- 深度技術評估 + 風險管理

### 時程導向專案

推薦：estimator + risk-analyst
- 精確估算 + 風險緩解

### 產品導向專案

推薦：ux-advocate + architect
- 體驗優先 + 技術支撐

### 全面分析

使用全部 4 視角（預設）

### 快速規劃

推薦：architect + estimator（--quick 模式）
- 核心設計 + 快速估算

## 深度模式額外視角

當使用 `--deep` 時，可添加：

| ID | 名稱 | 聚焦領域 |
|----|------|----------|
| `security` | 安全專家 | 認證授權、資料保護 |
| `ops` | 維運專家 | 部署、監控、擴展 |
