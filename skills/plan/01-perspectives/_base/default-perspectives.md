# Plan 視角定義

本 Skill 的視角定義集中管理於 `shared/perspectives/catalog.yaml`。

## 引用方式

```yaml
# 從 catalog.yaml 載入本 skill 適用的視角
perspectives:
  source: shared/perspectives/catalog.yaml
  filter:
    category: plan
  preset: standard  # 預設使用 4 視角
```

## 視角列表

### 標準模式 (4 視角)

| ID | 名稱 | 關注重點 | 模型層級 | 方法 |
|----|------|----------|----------|------|
| `architect` | 系統架構師 | 技術可行性和組件設計 | sonnet | deep |
| `risk-analyst` | 風險分析師 | 潛在風險和失敗場景識別 | sonnet | deep |
| `estimator` | 估算專家 | 工作量評估和優先順序 | haiku | deep |
| `ux-advocate` | UX 倡導者 | 使用者體驗和 API 設計 | haiku | deep |

### 快速模式 (2 視角)

| ID | 名稱 | 關注重點 |
|----|------|----------|
| `architect` | 系統架構師 | 技術可行性和組件設計 |
| `risk-analyst` | 風險分析師 | 潛在風險和失敗場景識別 |

### 深度模式 (6 視角)

標準 4 視角 + 跨領域視角：

| ID | 名稱 | 關注重點 |
|----|------|----------|
| `data-architect` | 資料架構師 | 資料庫設計、資料遷移、資料一致性 |
| `compliance-checker` | 合規檢查員 | 法規遵循和隱私保護 |

## 視角協作模式

```
┌─────────────────────────────────────────────────────────┐
│                    功能需求                              │
├─────────────────────────────────────────────────────────┤
│  技術層                          執行層                  │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐│
│  │ 系統     │  │ 風險     │  │ 估算     │  │ UX       ││
│  │ 架構師   │  │ 分析師   │  │ 專家     │  │ 倡導者   ││
│  │          │  │          │  │          │  │          ││
│  │ 技術設計 │  │ 風險識別 │  │ 工作量   │  │ 使用體驗 ││
│  │ 組件架構 │  │ 緩解策略 │  │ 優先排序 │  │ API 設計 ││
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘│
└─────────────────────────────────────────────────────────┘
```

## 視角組合策略

| 場景 | 推薦組合 | 說明 |
|------|----------|------|
| 技術導向專案 | architect + risk-analyst | 深度技術評估 + 風險管理 |
| 時程導向專案 | estimator + risk-analyst | 精確估算 + 風險緩解 |
| 產品導向專案 | ux-advocate + architect | 體驗優先 + 技術支撐 |
| 全面分析 | 全部 4 視角 | 預設模式 |
| 快速規劃 | architect + estimator | --quick 模式 |

完整定義請參考 [catalog.yaml](../../../../shared/perspectives/catalog.yaml)。
