# Default Perspectives

> 預設 4 視角配置：涵蓋功能與驗收的完整驗證框架

## 視角總覽

```
┌─────────────────────────────────────────────────────────┐
│                    功能驗證                              │
├─────────────────────────────────────────────────────────┤
│  功能層                          品質層                  │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐│
│  │ 功能     │  │ 邊界     │  │ 回歸     │  │ 驗收     ││
│  │ 測試員   │  │ 獵人     │  │ 檢查員   │  │ 驗證員   ││
│  │          │  │          │  │          │  │          ││
│  │ 核心功能 │  │ 極端輸入 │  │ 現有功能 │  │ 需求符合 ││
│  │ 使用流程 │  │ 錯誤處理 │  │ API 相容 │  │ DoD 驗證 ││
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘│
└─────────────────────────────────────────────────────────┘
```

## 視角 1: 功能測試員 (functional-tester)

### 角色定位

功能測試專家，專注於核心功能和使用者流程

### 驗證重點

- 核心功能正確性
- Happy Path 測試
- 使用者流程完整性
- 功能整合測試
- 狀態轉換正確性

### Prompt 模板

```
你是一位功能測試員。請驗證「{feature}」的核心功能：

驗證重點：
1. **核心功能**：主要功能是否正確運作？
2. **Happy Path**：正常使用流程是否順暢？
3. **狀態管理**：狀態轉換是否正確？
4. **整合點**：與其他功能的整合是否正確？

測試案例格式：
- 測試名稱
- 步驟
- 預期結果
- 實際結果
- 狀態（PASS/FAIL）
```

## 視角 2: 邊界獵人 (edge-case-hunter)

### 角色定位

邊界測試專家，專注於極端情況和錯誤處理

### 驗證重點

- 極端輸入
- 空值/Null 處理
- 邊界條件
- 錯誤處理
- 容錯能力
- 併發情況

### Prompt 模板

```
你是一位邊界獵人。請測試「{feature}」的邊界情況：

驗證重點：
1. **極端輸入**：超大、超小、負數、特殊字元
2. **空值處理**：null、undefined、空字串
3. **邊界條件**：最大值、最小值、臨界點
4. **錯誤處理**：異常是否正確處理？
5. **併發**：多重請求是否正確處理？

測試案例格式：
- 邊界類型
- 測試輸入
- 預期行為
- 實際行為
- 狀態（PASS/FAIL）
```

## 視角 3: 回歸檢查員 (regression-checker)

### 角色定位

回歸測試專家，專注於確保現有功能不受影響

### 驗證重點

- 現有功能完整性
- API 向後相容
- 整合點穩定性
- 效能基準
- 資料遷移正確性

### Prompt 模板

```
你是一位回歸檢查員。請驗證「{feature}」沒有破壞現有功能：

驗證重點：
1. **現有功能**：相關功能是否仍正常？
2. **API 相容**：API 契約是否保持？
3. **效能基準**：效能是否有明顯下降？
4. **資料相容**：現有資料是否正確處理？

回歸測試清單：
- 功能名稱
- 測試前狀態
- 測試後狀態
- 狀態（PASS/FAIL/REGRESSION）
```

## 視角 4: 驗收驗證員 (acceptance-validator)

### 角色定位

驗收測試專家，專注於需求符合度和 DoD

### 驗證重點

- 需求符合度
- Definition of Done
- 使用者故事驗證
- 業務邏輯正確性
- 文檔完整性

### Prompt 模板

```
你是一位驗收驗證員。請驗證「{feature}」符合驗收標準：

驗收標準：
{acceptance_criteria}

驗證重點：
1. **需求符合**：是否實現了所有需求？
2. **DoD 檢查**：是否符合 Definition of Done？
3. **業務邏輯**：業務規則是否正確實現？
4. **文檔**：是否有必要的文檔？

驗收清單：
- 驗收標準
- 驗證方式
- 結果
- 狀態（PASS/FAIL）
```

## 視角組合策略

### 快速驗證

推薦：functional-tester + acceptance-validator（--quick 模式）
- 核心功能 + 驗收確認

### 安全敏感

推薦：edge-case-hunter + regression-checker
- 邊界測試 + 回歸確認

### 發布驗證

使用全部 4 視角（預設）

## 深度模式額外視角

當使用 `--deep` 時，可添加：

| ID | 名稱 | 聚焦領域 |
|----|------|----------|
| `performance` | 效能測試員 | 負載測試、壓力測試 |
| `security` | 安全測試員 | 滲透測試、漏洞掃描 |
